# 流程总览

所有平台级流程已代码化，位于 `suri-agent/process/` 下。角色内部流程由角色在 `group/<role>/` 中自行定义。

---

- **"标准任务调度流"** 针对程序调度。suri 接收需求 → 解析并匹配部门 → 下发总监 → 成员执行 → 总监审核 → suri 汇总交付用户。

- **"跨部门协作流"** 针对程序调度。需求方总监向提供方总监发起私聊请求（抄送 suri），双方各自在部门内分派，每 30 分钟同步进度至 suri，suri 整合后交付用户。

- **"异常处理与升级流"** 针对程序调度。成员失败 → 重试（最多 3 次）→ 汇报总监 → 总监无法解决则汇报 suri → suri 向用户汇报并请求决策。

- **"用户决策回流"** 针对程序调度。开发人员遇到无法自主解决的问题 → 总监尝试解决 → suri 整理上下文和可选方案 → 回流给用户判断 → 用户给出决策指令 → 开发人员继续执行。

- **"能力缺口处理流"** 针对程序调度。需求无法匹配任何部门时 → suri 向用户展示能力边界 → 用户确认是否需要新建部门/角色 → suri 向 suri-hr 发起组织扩展请求 → 新角色上线后重新调度原需求。

- **"配置变更审批流"** 针对程序与调度。操作者生成变更报告 → 确认是控制角色 → security_admin 审核 → suri 向用户请求确认 → 用户回复"是" → 执行修改 → git_admin 记录变更日志。

- **"首次运行初始化流"** 针对程序启动。用户首次运行程序时，引导配置模型参数（模型选择、API Key、端点地址），完成后可选是否连接 Telegram 机器人（输入 Token 即连接，跳过则仅使用 TUI 终端交互）。初始化完成后写入 config.yaml 和 .env，下次启动直接读取，不再重复引导。

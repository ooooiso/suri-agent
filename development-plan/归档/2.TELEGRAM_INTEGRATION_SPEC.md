
# Suri Telegram 通信集成 — 需求汇总

> 版本: 1.0  
> 日期: 2026-04-30  
> 来源: 整合用户与 AI 的全部对话  
> 范围: **仅 Telegram**，不包含飞书/其他渠道

---

## 1. 需求背景

将 Suri 平台与 Telegram Bot 对接，实现：
- 用户通过 Telegram 与 suri 交互（发起任务）
- 角色工作成果投影到 Telegram 群组展示
- suri-hr 创建角色时同步配置 Telegram 群组

---

## 2. 核心需求（从对话提取）

### 2.1 Bot 连接
- suri 作为 Telegram Bot 运行，接收私聊和群组消息
- 使用 `python-telegram-bot` 或 `aiogram` 实现真连接（替换当前占位符）

### 2.2 用户输入入口
- **私聊**：用户直接给 Bot 发消息 → 转为任务交给 suri
- **群组@Bot**：在绑定群里 @Bot 发消息 → 同样转为任务

### 2.3 指派 HR 创建角色 + 配置群组
- 用户私聊 suri："创建一个设计部"
- suri 调度给 suri-hr
- suri-hr：
  1. 调用 `RoleBuilder.create_role()` 创建角色目录
  2. 输出指引：让用户在 Telegram 创建群组、拉 Bot 入群、发送 `/bind_group design`
  3. 记录群组映射关系

### 2.4 群组展示通信记录
- 角色内部通信完成后，选择性同步到 Telegram 群组
- 同部门通信 → 投影到该部门群
- 跨部门通信 → 投影到双方部门群 + 中枢群
- **投影是单向展示，不是通信本身**

### 2.5 通信规则驱动
- 角色间真正通信在内部完成（`Messenger` + `CommunicationRule`）
- Telegram 不参与角色间通信路由，只接收投影
- 跨部门通信**不抄送 suri**，总监级直连

### 4.2 通信规则修改要点

```python
# rules/communication.py

def execute(self, context):
    return {
        "allowed": self._check_allowed(...),
        "copy_to": [],                    # 跨部门不抄送 suri
        "project_to": [                   # 投影目标（Telegram 群别名）
            "tg:design", "tg:central"
        ],
        ...
    }
```

### 4.3 Messenger 修改要点

```python
# role/messenger.py

async def send(self, sender, receiver, message):
    # 原有：规则校验 + 内部投递
    policy = self.rule_engine.execute('communication_protocol', ...)
    self.memory.save_message(receiver, message)
    
    # 新增：投影到 Telegram
    if self.projection and policy.get('project_to'):
        for ref in policy['project_to']:
            if ref.startswith('tg:'):
                await self.projection.project_message(ref, sender, receiver, message)
    
    return {'success': True}
```

## 5. 数据流

### 5.1 用户发起任务

```
用户 Telegram 私聊 "帮我设计 Logo"
    ↓
Telegram Bot 接收 → 组装 StandardMessage
    ↓
suri 处理 → TaskService.dispatch() → designer
    ↓
designer 完成 → messenger.send(designer, suri, result)
    ↓
投影触发 → 设计部群显示结果
    ↓
suri 私聊回复用户 "已完成"
```

### 5.2 角色内部通信（投影到群）

```
designer → engineer（内部 Messenger）
    ↓
CommunicationRule.project_to = ["tg:design", "tg:engineering", "tg:central"]
    ↓
ProjectionService 格式化 → Telegram 发送
    ↓
设计群 / 工程群 / 中枢群 同时展示
```

## 7. 测试验证

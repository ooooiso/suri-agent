# Web UI 仪表盘规格说明书

> 版本: 1.0.0
> 日期: 2026-05-01
> 状态: 待开发

---

## 1. 设计目标

在网页中可视化呈现：
- **角色任务状态** — 谁在执行什么任务，进度如何
- **角色机能能力** — 每个角色的 Soul、技能、模型、工具权限
- **部门指派关系** — 组织架构树、调度路径、跨部门协作

同时提供**跨平台桌面程序**封装能力（Tauri）。

---

## 2. 架构原则

### 2.1 物理隔离

```
ui/                          ← UI 所有内容统一在此（可一键删除）
├── index.html
├── css/
├── js/
├── server.py                ← 轻量 HTTP server（静态文件 + /rpc 代理 + /events SSE）
└── ui.md

suri-agent/webui/            ← 后端 API 扩展（最小化）
├── __init__.py
└── api_bridge.py            ← UI 专用 API 方法注册

suri-agent/                  ← 核心代码不动（只修 API bug）
```

### 2.2 零侵入
- 不修改现有 `suri-agent/access/tui/cli.py`
- 不修改现有 `suri-agent/access/tui/server.py` 核心逻辑（只新增可选 SSE 路由）
- 前端零框架依赖（纯 HTML/CSS/JS）
- 通信单一入口：前端 → `ui/server.py` → 代理到现有 `:8080/rpc`

### 2.3 可插拔
```bash
rm -rf ui/ suri-agent/webui/   ← 一键删除
# suri-agent CLI/Telegram/JSON-RPC 完全不受影响
```

## 3. 跨平台 UI 框架选型

| 方案 | 体积 | 性能 | 用户侧依赖 | 推荐度 |
|------|------|------|-----------|--------|
| **纯 Web (HTML/CSS/JS)** | 零 | 浏览器级 | 无 | ⭐⭐⭐⭐⭐ |
| **Tauri (Rust+Webview)** | ~5-15MB | 原生 | macOS/Win 零依赖；Linux 需 webkit2gtk | ⭐⭐⭐⭐ |
| PyWebView | ~30MB | 一般 | Python 依赖 | ⭐⭐⭐ |
| Electron | ~100MB | 好 | 无 | ⭐⭐ |

**最终策略：纯 Web Dashboard 为核心 + Tauri 可选封装**

- 浏览器打开即用
- Tauri 将同一套前端代码打包为 Windows/Mac/Linux 桌面程序（~5MB）
- 无需 Flask/FastAPI — 静态文件直接由 `ui/server.py` 或 `python -m http.server` 提供

## 4. 功能模块设计

### 4.1 Dashboard 首页

- 系统状态卡片：角色数、规则数、运行时长
- 活跃任务统计（pending / in_progress / completed / failed 各多少）
- 模型池状态（当前默认模型、可用模型数）
- 最近 5 条系统日志

### 4.2 任务看板（Kanban）

- 四栏：待处理 / 进行中 / 已完成 / 失败
- 任务卡片：任务ID、发起人、目标角色、创建时间、最新消息预览
- 筛选：按角色 / 按部门 / 按时间范围 / 按关键词搜索
- 点击展开：任务详情 + 完整消息历史

### 4.3 角色面板

- 角色卡片网格：颜色标识、名称、类型、部门
- 点击展开详情：
  - Soul 原文（来自 `group/central/<role>/<role>.md`）
  - 技能列表（`skills/skills.md` 解析）
  - 模型配置（当前使用哪个模型）
  - 工具权限（可调用哪些工具）
  - 记忆统计（insights 数量、messages 数量）
  - 近期任务（该角色参与的最新 10 个任务）

### 4.4 组织架构图

- 树形结构：`central` → `suri` / `suri-hr` / `suri-dev` / `document-review`
- 点击部门/角色查看详情面板
- 可视化调度路径：用户请求 → suri → 目标角色 → 结果回流

### 4.5 审批中心

- 待审批列表（变更报告、文件修改请求）
- 一键 Approve / Reject
- 审批历史记录

### 4.6 交互终端

- 网页版聊天框，直接与 suri 对话
- 保留对话历史（按 session 隔离）
- 支持 / 命令（/help、/roles、/status 等）

### 4.7 实时推送（可选）

- SSE `/events` 长连接
- 任务状态变更自动刷新看板
- 新消息到达显示通知
- 审批请求弹出提示

**如果不做 SSE**：前端每 5 秒轮询一次，效果等同。

## 5. 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | HTML5 / CSS3 / Vanilla JS | 无框架、无 npm、无构建步骤 |
| 图表 | CSS Grid / Flexbox + SVG | 纯原生，无需外部库 |
| 通信 | JSON-RPC over HTTP | 复用现有 API |
| 实时 | SSE (Server-Sent Events) | 可选，不强制 |
| 桌面封装 | Tauri | 可选 Phase 4 |

## 7. API 设计

### 7.1 新增 RPC 方法（由 api_bridge.py 注册）

```
suri.getDashboardStats              → 聚合统计（活跃任务数、角色数、模型状态）
suri.getAllTasks                    → 遍历所有 role.db 聚合任务列表
suri.getTaskById(task_id)           → 跨所有角色数据库搜索指定任务
suri.getPendingApprovals            → 实际查询 approvals 表 pending 状态
suri.getDepartmentTree              → 返回部门-角色层级结构
```

### 7.2 现有 RPC 方法修复

```
suri.get_tasks()                    → 从空列表改为聚合查询
suri.get_task_detail(task_id)       → 修复 role_id 缺失
suri.get_task_messages(task_id)     → 修复 role_id 缺失
suri.get_pending_approvals()        → 从空列表改为实际查询
```

### 7.3 SSE 事件类型（可选）

```
task:created        → 新任务创建
task:status_changed → 任务状态变更
message:received    → 新消息到达
approval:pending    → 新审批请求
```

## 9. 启动方式

```bash
# 终端1：启动 suri-agent JSON-RPC API（现有，不动）
python suri-agent/access/tui/server.py --port 8080

# 终端2：启动 UI 服务（新增）
python ui/server.py --port 3000
# 或更简单：python -m http.server 3000 --directory ui/

# 浏览器打开
open http://localhost:3000
```

## 10. 风险与回滚

| 风险 | 缓解措施 |
|------|----------|
| UI 做坏了影响核心 | UI 是只读消费层，不持有状态、不写数据库。最坏情况页面显示异常 |
| API 改造引入 bug | 只新增方法/修 TODO，不动现有逻辑。新增方法放在 webui/api_bridge.py，做坏了删掉即可 |
| 前端代码膨胀 | 纯 HTML/CSS/JS，无构建步骤，无 node_modules，可控 |

**回滚方式：**
```bash
rm -rf ui/ suri-agent/webui/
# 主程序完全恢复
```

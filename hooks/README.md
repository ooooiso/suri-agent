# 事件钩子目录

## 说明

本目录由 `ops_admin` 管理，存放平台运行时的事件钩子脚本。

## 当前钩子

| 钩子文件 | 触发时机 | 功能 | 管理角色 |
|---------|---------|------|---------|
| `pre_file_change.py` | 任何文件修改前 | 读取 `file_ownership.md` 和 `security.md`，校验权限与审批状态 | ops_admin |

## 钩子开发规范

1. 钩子脚本必须是无副作用的纯检查函数，返回 `allow` 或 `deny`。
2. 拒绝时必须附带明确的原因代码。
3. 所有钩子操作记录到 `logs/hook.log`。

> 新增钩子需 `ops_admin` 发起，经 `security_admin` 审核后部署。

# Soul 进化（Soul Evolution）

> 定义 suri-agent 中角色灵魂（Soul）的进化机制。Soul 是角色的自我定义，进化意味着角色认知和能力的升级。

---

## 一、Soul 进化的触发条件

| 触发条件 | 触发方 | 说明 |
|---------|--------|------|
| 角色自我反思 | 角色自身 | 角色完成任务后分析自身不足，提出 Soul 更新建议 |
| 用户反馈 | 用户 | 用户明确指出角色行为需要调整 |
| suri 评估 | suri 角色 | suri 发现角色的能力边界与实际需求不匹配 |
| role_learner 累积 | role_learner | 通过长期观察发现角色的表现与现有不足 |
| 外部冲突 | 协作角色 | 协作中频繁冲突，需要调整角色边界 |

## 二、Soul 进化流程

```
触发条件满足
    │
    ▼
生成 Soul 更新草案（LLM 辅助）
    │
    ▼
发布 role.soul_updating 事件
    │
    ▼
通知所有相关方（协作角色、suri）
    │
    ▼
suri 评估变更影响
    │
    ├── 向后兼容 → 自动审批
    └── 不兼容 → 需要用户确认
    │
    ▼
用户确认 / suri 审批
    │
    ▼
更新 soul.md 文件
    │
    ▼
发布 role.soul_updated 事件
    │
    ▼
相关方重新加载角色上下文
```

## 三、Soul 变更影响评估

| 变更类型 | 影响范围 | 是否需要用户确认 |
|---------|---------|----------------|
| 能力描述修改 | 仅本角色 | 否（suri 审批即可） |
| 新增职责 | 本角色 + 协作角色 | 是 |
| 权限变更 | 本角色 + security_service | 是 |
| 角色关系修改 | 所有相关角色 | 是 |
| 能力边界缩减 | 本角色 | 否 |

## 四、Soul 版本与回滚

```
Soul 文件路径：roles/{role_id}/soul.md（单一文件，非版本化存储）

变更记录路径：roles/{role_id}/memories/changelogs/soul_changelog.md

变更日志格式：
  | 时间 | 变更摘要 | 触发条件 | 审批人 |
  |------|---------|---------|--------|

回滚操作：
  1. 从 git 历史恢复旧版本 soul.md
  2. 发布 role.soul_updated 事件（版本指向 git commit hash）
  3. 所有相关方重新加载
```

## 五、Soul 进化最佳实践

1. **小步迭代**：每次只修改一个方面，便于影响评估和回滚
2. **充分通知**：变更前通知所有协作角色，收集反馈
3. **测试验证**：变更后执行技能测试，确保不破坏现有能力
4. **文档同步**：变更同步更新 role_manager 的能力索引

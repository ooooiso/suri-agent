## 1. 模块定位

自学习模块是 Suri 平台的经验积累与进化系统：

- **角色自学习**：每个业务角色在执行任务后，自动复盘提取经验，沉淀到角色私人记忆区。下次同类任务调度时，将相关经验注入 System Prompt
- **主程序自学习**：suri 核心调度器汇总全平台任务数据，持续优化部门匹配策略、用户画像、模型路由偏好

**设计哲学**：
1. 异步非阻塞 — 学习过程在后台进行，不延迟用户响应
2. 文件即记忆 — 经验以 Markdown 形式存储，AI 可直接读写理解
3. 分层防御冗余 — 写入去重 + 夜间压缩 + 动态遗忘 + 注入截断
4. 角色自治 — 角色 A 的经验不可泄露给角色 B

## 2. 数据流

```
TaskService.dispatch() 成功返回前
    └── asyncio.create_task(_trigger_learning(task_id, role_id))
        └── RoleLearner.learn_from_task() [后台异步]
            ├── FeedbackCollector.collect_task_feedback()
            ├── ExperienceExtractor.extract() (调用 LLM)
            ├── 去重检查 (_is_duplicate)
            ├── 写入经验文件 (MemoryService.save_role_insight)
            └── 记录日志 (LoggerService.log_learning)

下次同角色调度时
    └── ContextService.build_context()
        └── _get_learning_insights() 注入最近相关经验
```

## 5. 角色自学习引擎（RoleLearner）

### 5.1 需求

- ✅ **端到端学习流程** — `learn_from_task()`：收集反馈 → 提取经验 → 去重 → 写入文件 → 技能建议
- ✅ **去重检查** — `_is_duplicate()` 使用标题杰卡德相似度 > 0.6 或 key_point 5-10 字短语匹配
- ✅ **经验合并** — `_merge_insight()` 更新 trigger_count、last_triggered、追加验证记录
- ✅ **新经验保存** — `_save_insight()` 调用 `MemoryService.save_role_insight()`
- ✅ **技能建议检查** — `_check_skill_update()` 识别 tool_pattern 和 skill_suggestion
- ✅ **错误处理** — 学习失败不影响系统运行，记录错误日志

### 5.2 存储结构

- **路径**: `group/<dept>/<role>/memories/insights/YYYYMMDD_HHMMSS_{sanitized_title}.md`
- **格式**: YAML frontmatter（insight_id, category, created_at, trigger_count, confidence, ttl_days, last_triggered）+ Markdown body（情境/要点/避免/验证记录）

## 1. 工具定义与边界

### 1.1 工具（Tool）— 通用基础设施

- **归属**：平台公共，由 `suri_dev`（maintainer）维护
- **位置**：`suri-agent/tools/<tool_id>/`
- **用途**：所有角色均可调用的通用能力
- **管理**：`suri_dev` 负责工具的开发、维护、版本升级
- **调用**：角色通过 `ToolService` 调用工具，受 `tools:` 白名单权限控制

### 1.2 技能（Skill）— 角色专属能力

- **归属**：角色私有，由角色自身维护
- **位置**：`group/<dept>/<role>/skills/<skill_id>/`
- **用途**：角色专属的能力模块，可调用工具来完成特定任务
- **内容**：`skill.md`（定义）+ 可选的 `script.py` + `resources/`
- **调用**：角色在执行任务时，根据需求匹配自身技能

### 1.3 工具 vs 技能

| | Tool | Skill |
|--|------|-------|
| 归属 | 平台公共 | 角色私有 |
| 维护者 | `suri_dev` | 角色自身 |
| 位置 | `suri-agent/tools/` | `group/<role>/skills/` |
| 调用范围 | 所有有权限的角色 | 仅所属角色 |
| 内容 | 通用函数/接口 | 业务流程 + 脚本 + 资源 |

## 2. 公共工具清单

### 2.1 file_read — 文件读取

- ✅ **功能**：支持相对路径（基于 project_root）和绝对路径，返回内容和路径
- ✅ **位置**：`suri-agent/tools/file_read/scripts/main.py`
- ✅ **权限**：public

### 2.2 file_write — 文件写入

- ✅ **功能**：自动创建父目录，返回路径和大小
- ✅ **位置**：`suri-agent/tools/file_write/scripts/main.py`
- ✅ **权限**：需 SecurityService 审批令牌（注释提到"由调用方验证"）

### 2.3 file_list — 文件列表

- ✅ **功能**：返回 files 和 directories 两个列表，自动过滤隐藏文件
- ✅ **位置**：`suri-agent/tools/file_list/scripts/main.py`
- ✅ **权限**：public

### 2.4 shell_exec — Shell 执行

- ✅ **功能**：命令安全过滤 + 子进程执行
- ✅ **安全过滤**：禁止 rm -rf /、mkfs、dd、curl/wget、shutdown、sudo 等危险模式
- ✅ **超时控制**：`subprocess.run()` 带超时
- ✅ **位置**：`suri-agent/tools/shell_exec/scripts/main.py`
- ✅ **权限**：maintainer

### 2.5 db_query — 数据库查询

- ✅ **功能**：只读 SQL 查询，以 `mode=ro` 打开数据库，返回字典列表
- ✅ **位置**：`suri-agent/tools/db_query/scripts/main.py`
- ✅ **权限**：public
- 🔄 **待完善**：SQL 注入防护（直接执行传入的 query 字符串）

### 2.6 db_insert — 数据库插入

- ✅ **功能**：SQLite 数据插入
- ✅ **位置**：`suri-agent/tools/db_insert/scripts/main.py`
- ✅ **权限**：public
- 🔄 **待完善**：SQL 注入防护（使用字符串拼接表名和列名，无白名单校验）

### 2.7 model_manager — 模型管理工具

- ✅ **功能**：
  - `action=list` — 按类型分组返回所有模型，含能力标签和成本等级
  - `action=switch` — 修改默认模型并持久化
  - `action=classify` — 按预设规则重新分类并写入配置
  - `action=generate_docs` — 输出可用模型配置 Markdown
- ✅ **位置**：`suri-agent/tools/model_manager/scripts/main.py`
- ✅ **权限**：public
- 🔄 **待完善**：未提供 delete 操作

### 2.8 web_fetch — 网页获取

- ✅ **功能**：
  - `action=fetch` — 使用 httpx 获取网页，提取纯文本，限制最大长度（默认 8000）
  - `action=search` — 使用 DuckDuckGo 文本搜索 API
- ✅ **安全校验**：禁止 file:// 等本地协议，URL 格式校验
- ✅ **位置**：`suri-agent/tools/web_fetch/scripts/main.py`
- ✅ **权限**：public
- 🔄 **待完善**：仅获取静态 HTML，无 JavaScript 执行

## 3. 工具权限体系

### 3.1 需求

- ✅ **Soul 白名单** — 角色 Soul 中 `tools` 字段显式授权的工具
- ✅ **public 权限** — 所有角色可用
- ✅ **maintainer 权限** — 仅 `suri_dev` 等 maintainer 类型角色可用
- ✅ **特定 role 权限** — 仅指定 role_id 可用
- ✅ **参数校验** — `validate_params()` 从 tool.md frontmatter 的 `params` 字段解析定义，做必填和类型检查
- ✅ **调用记录** — `_record_tool_call()` 统一通过 `LoggerService` 写入 `logs/tool_calls/`
- ✅ **权限矩阵一致性** — 测试覆盖 6 组角色+工具权限组合

### 3.2 角色 Soul tools 字段示例

```yaml
tools:
  - file_read
  - file_list
  - db_query
```

## 5. MCP 框架

### 5.1 架构设计

- ✅ **服务注册/注销** — `MCPRegistry.register()`、`unregister()`，自动发现工具
- ✅ **角色工具绑定** — `bind_tool_to_role()`、`get_role_tools()`
- ✅ **工具执行** — `execute_tool()` 分派到对应服务
- ✅ **自动增强** — `auto_enhance_role()` 按任务需求关键词匹配推荐工具
- ✅ **上下文注入** — `inject_context()` 将 MCP 工具说明追加到角色系统提示
- ✅ **数据类定义** — `MCPTool`、`MCPServer`
- ✅ **抽象接口** — `discover_tools()`、`execute()`、`get_server_info()`

### 5.2 MCP 服务实现状态

| 服务 | 状态 | 说明 |
|------|------|------|
| CodeExecution | 🔄 预留 | `exec_python`、`exec_shell` 定义存在，execute 返回"尚未实现沙箱环境" |
| Filesystem | 🔄 部分实现 | `fs_read`、`fs_list`、`fs_search` 只读操作实现，无写入/删除 |
| WebSearch | 🔄 预留 | `web_search`、`web_fetch` 定义存在，execute 返回空结果 |

### 5.3 待实现

- 🔄 **代码执行沙箱** — 隔离环境执行 Python/Shell 代码

## 6. 技能管理流程

### 6.1 需求

- ✅ **角色发起** — 角色在自身 `skills/` 目录下创建技能文件夹和说明文档
- ✅ **用户同意** — 涉及外部调用、文件系统操作、网络访问的技能，必须获得用户明确授权
- ✅ **上报 hr** — 角色将新技能信息上报 suri_hr
- ✅ **hr 同步** — suri_hr 更新 `group_function.md` 和角色能力清单
- ✅ **document-review 审核** — 涉及安全敏感操作的技能需审核
- ✅ **技能模板** — `group/central/suri-hr/skills/templates/role_skills.md` 含 `tools` 字段和调用方式

### 6.2 技能-工具-任务闭环

- ✅ **技能模板增加 tools 字段** — 明确技能依赖的工具
- ✅ **ExperienceExtractor 提取工具使用模式** — 从任务记录中提取工具调用模式
- ✅ **RoleLearner 检测技能更新建议** — 识别 tool_pattern 和 skill_suggestion
- ✅ **ToolSyncRule 检查技能依赖** — 确保技能依赖的工具存在
- ✅ **ToolService 记录工具调用历史** — 用于后续分析和优化

# 冲突解决

> 定义角色间冲突的识别和解决机制。

---

## 一、冲突类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **资源冲突** | 多个角色争用同一资源 | 都想写同一个文件 |
| **决策冲突** | 角色对同一问题有不同意见 | 前后端接口方案分歧 |
| **权限冲突** | 角色尝试越权操作 | 工作角色试图管理系统插件 |

## 二、解决策略

### 1. suri 仲裁（默认）

```
角色A vs 角色B → 升级给 suri → suri 仲裁 → 双方接受
                                           ↓
                                       拒绝 → 上报用户
```

### 2. 项目总监仲裁（如存在）

```
角色A vs 角色B → 升级给 project_director → 总监决策 → 执行
```

## 三、升级路径

```
角色间冲突
    │
    ├─ 有项目总监？ → 项目总监仲裁
    │
    ├─ 无项目总监？ → 升级给 suri
    │
    └─ suri 无法解决？ → 上报用户决策

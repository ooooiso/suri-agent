# 技能组合与编排

> 定义角色如何组合多个技能完成复杂任务。

---

## 一、技能组合方式

### 1. 串行组合

```
技能 A → 技能 B → 技能 C
示例：读取代码 → 分析代码 → 生成优化建议
```

### 2. 并行组合

```
技能 A ──┐
技能 B ──┤→ 汇总结果
技能 C ──┘
示例：前端+后端+数据库 并行开发
```

### 3. 条件组合

```
技能 A
  ├─ 成功 → 技能 B
  └─ 失败 → 技能 C
示例：尝试自动修复 → 如果失败 → 申请人工介入
```

## 二、角色技能编排

角色通过 LLM 自主编排技能调用序列，无需预先定义：

```
角色接收任务
    │
    ▼
角色分析任务 → 确定需要的技能
    │
    ▼
角色调用技能序列（LLM 动态决定顺序）
    ├─ skill_1.analyze()
    ├─ skill_2.execute()
    └─ skill_3.validate()
    │
    ▼
角色整合结果 → 返回用户
```

## 三、跨角色技能组合

当单个角色技能不足时，suri 协调多角色：

```
用户需求（需要前端+后端+数据库）
    │
    ▼
suri 分解任务
    ├─ 前端部分 → frontend_dev
    ├─ 后端部分 → backend_dev
    └─ 数据库部分 → data_analyst
    │
    ▼
角色们各自执行 → 结果汇总 → 返回用户

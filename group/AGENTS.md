# group/ AGENTS.md — 角色层强制规则

> 本文件对 `group/` 目录及所有子目录（所有角色）具有约束力。
> 优先级：用户指令 > group/AGENTS.md > 根目录 AGENTS.md > 通用最佳实践

---

## 绝对规则

### 1. 所有角色更改代码后必须同步更新文档

无论哪个角色（suri-dev、suri-hr、业务角色或未来新增角色），**修改代码后必须触发文档同步流程**。

**标准流程：**
```
角色完成代码变更
    → 自检：是否影响文档？（参照根目录 .kimi/checklists/doc-sync.md）
    → 若影响：更新对应 .md 文件
    → 提交给 document-review 审核（或记录待审核标记）
    → 审核通过后完成闭环
```

**不更新文档的代码变更视为未完成，document-review 有权驳回。**

### 2. document-review 是文档同步机制的守门人

document-review 角色的核心职责升级为：
- **监督**：所有角色的代码变更是否同步更新了文档
- **审查**：检查文档与代码的一致性、完整性、时效性
- **强制**：发现未同步的，生成审核报告要求角色补文档
- **记录**：维护文档同步历史，追踪遗漏项

### 3. 代码文件必须标注关联文档

所有角色在创建或修改代码文件时，应在文件头部的模块 docstring / 注释中标注关联文档路径。这是为了方便任何后续接手的模型快速定位需求来源。

格式：
```python
"""
关联文档: <相对路径>
文档同步提醒: 修改本文件后，请检查并同步更新关联文档。
"""
```

## 角色职责补充

| 角色 | 文档相关职责 |
|------|-------------|
| **suri-dev** | 修改 suri-agent/ 核心代码后，同步更新对应模块 .md 和 wiki/ 文档 |
| **suri-hr** | 新建/修改角色后，同步更新角色 Soul 文件、group_function.md、模板文件 |
| **document-review** | 审核所有角色的文档更新，确保代码-文档一致性，监督机制执行 |
| **业务角色** | 修改自身技能脚本或配置后，同步更新角色目录下的说明文档 |
| **任何未来新增角色** | 进入项目即默认遵守本规则，suri-hr 在角色初始化时注入此约束 |

---

## 执行检查点

suri-hr 在创建新角色时，必须在角色的初始化模板中包含：
1. 本规则的引用或摘要
2. 角色专属文档路径清单
3. document-review 的审核对接方式

document-review 定期（或按需）执行：
1. 扫描最近变更的代码文件
2. 检查对应文档是否同步更新
3. 生成不一致报告，提交给 suri 调度处理
